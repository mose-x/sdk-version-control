# SVC Shims + .svcrc 方案

## 一、背景与痛点

| 平台 | 现状问题 |
|------|---------|
| **macOS/Linux** | 往 `.zshrc` / `.bashrc` / `.zshenv` 等多个文件里塞 `source` 行，数量多、清理难，用户的 shell 配置越来越乱 |
| **Windows** | 每个 SDK 往注册表里写一行 PATH，装 100 个 SDK 就有 100 行，迁移安装目录时全部要改 |
| **共同问题** | SVC 改了什么、改在哪了，用户不清楚；想卸载时不知道从哪清理 |

---

## 二、核心方案：Shims 模式

### 2.1 原理
在 `~/.svc/shims/` 目录下为每个 SDK 生成一个**垫片程序（shim）**，PATH 里只放这一个目录。

调用 `go` / `java` / `node` 等命令时，先经过 shim，shim 读取配置找到当前激活的版本，再转发到真实的可执行文件。

### 2.2 目录结构

```
~/.svc/
├── shims/               ← PATH 唯一入口（永远只有这一行）
│   ├── go
│   ├── java
│   ├── javac
│   ├── node
│   ├── npm
│   ├── python
│   ├── pip
│   ├── mvn
│   ├── gradle
│   └── ...
├── go/
│   ├── 1.22.0/
│   ├── 1.21.0/
│   └── ...
├── jdk/
│   ├── 17.0.9/
│   └── ...
├── python/
│   ├── 3.12.0/
│   └── ...
├── config.json          ← 记录各 SDK 当前激活版本
└── ...
```

### 2.3 优势

| 对比项 | 现在 | Shims 模式 |
|--------|------|-----------|
| PATH 行数 | 每个 SDK 一行（N 行） | **永远 1 行** |
| 迁移难度 | N 处要改 | **1 处** |
| 版本切换 | 改 PATH | **改 config.json** |
| 冲突风险 | 多个版本路径都在 PATH | **只有 shims 在 PATH** |

## 三、macOS / Linux：`.svcrc` 单文件模式

### 3.1 设计

用户的 shell 配置文件里**只加一行**：

```bash
# ~/.zshrc（或 ~/.bashrc）
[[ -f ~/.svcrc ]] && source ~/.svcrc
```

`~/.svcrc` 由 SVC 自动维护，内容包括：

```bash
# ~/.svcrc — 由 SVC 自动管理
export PATH="$HOME/.svc/shims:$PATH"
export JAVA_HOME="$HOME/.svc/jdk/17.0.9"
export GOROOT="$HOME/.svc/go/1.22.0"
export ANDROID_HOME="$HOME/.svc/android"
```

### 3.2 Shell 配置策略

| Shell | 配置文件 | 加的内容 |
|-------|---------|---------|
| zsh | `~/.zshrc` | `[[ -f ~/.svcrc ]] && source ~/.svcrc` |
| bash (Linux) | `~/.bashrc` | 同上 |
| bash (macOS) | `~/.bash_profile` | 同上 |
| fish | `~/.config/fish/conf.d/svc.fish` | `test -f ~/.svcrc.fish; and source ~/.svcrc.fish` |

**原则：默认只改用户默认 shell 的一个配置文件，其他的由用户手动添加。**

---

## 四、Windows：Shims + 注册表

### 4.1 设计
PATH 里只有一行：
```
C:\Users\you\.svc\shims
```

Shims 形式：
- **方式 A**：每个 shim 是一个 `.cmd` 批处理文件
- **方式 B**：用 Go 写一个通用 shim 程序，通过硬链接/符号链接区分命令

环境变量（`JAVA_HOME` 等）仍写注册表，但因为少了 PATH 的 N 行，整体简洁很多。

## 五、透明化配置：设置页「Shell 环境」面板

### 5.1 功能清单

| 功能 | 说明 |
|------|------|
| **检测默认 Shell** | 读 `$SHELL` 自动识别 |
| **已配置列表** | 展示哪些配置文件里加了 SVC 的行 |
| **单个移除** | 从指定文件里删除 SVC 相关行 |
| **添加到其他 Shell** | 用户手动选择要加到哪个 shell |
| **一键移除所有** | 从所有文件里清除 SVC 配置 |
| **打开文件** | 点击路径直接用系统编辑器打开 |
| **.svcrc 预览/编辑** | 查看和修改 `.svcrc` 内容 |
| **重新生成 .svcrc** | 根据当前激活版本重新生成 |

### 5.2 界面大致结构

```
┌─────────────────────────────────────────────┐
│  Shell 环境配置                               │
├─────────────────────────────────────────────┤
│                                              │
│  📍 当前默认 Shell：zsh                      │
│                                              │
│  ✅ 已配置的 Shell 配置文件：                 │
│    ┌──────────────────────────────────────┐  │
│    │  ~/.zshrc                [移除]      │  │
│    │  ~/.bashrc               [移除]      │  │
│    └──────────────────────────────────────┘  │
│                                              │
│  ➕ 添加到其他 Shell：                        │
│    [zsh      ▼]  [添加]                      │
│    [bash     ▼]  [添加]                      │
│    [fish     ▼]  [添加]                      │
│    [zshenv   ▼]  [添加]                      │
│                                              │
│  📄 ~/.svcrc 配置文件：                       │
│    [打开文件]  [重新生成]  [重置]             │
│    ┌──────────────────────────────────────┐  │
│    │ export PATH="$HOME/.svc/shims:$PATH" │  │
│    │ export JAVA_HOME="..."               │  │
│    └──────────────────────────────────────┘  │
│                                              │
│  🗑️  一键移除所有 SVC Shell 配置               │
│                                              │
└─────────────────────────────────────────────┘
```

## 六、设计原则

1. **单一入口原则**：每个 shell 配置文件里，SVC 只加一行 `source ~/.svcrc`
2. **权责分离**：SVC 只管 `~/.svcrc` 内容，不碰用户 shell 配置里的其他东西
3. **完全透明**：改了哪个文件、加了什么，用户都能看到
4. **可完全撤销**：一键移除所有 SVC 痕迹，用户配置恢复原样
5. **最小侵入**：默认只改一个文件，其他由用户主动添加
